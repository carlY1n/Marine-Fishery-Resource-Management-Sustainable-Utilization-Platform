# 🌊 海洋渔业资源管理与可持续利用平台

> B端管理系统 & 数据密集型场景

## 项目简介

本平台是一套面向海洋渔业资源全生命周期管理的数据库驱动 Web 系统，聚焦"监测—管理—决策"三位一体的业务闭环。系统服务于四类角色：普通用户、渔业从业人员、海洋资源管理人员与海洋研究人员，通过严格的权限分层实现精细化的数据治理。

在全球渔业资源持续衰减、海洋生态系统面临不可逆损耗的背景下，本平台以大数据技术为手段，将复杂的海洋环境信息与渔业资源信息结构化、可视化，为渔业从业者、科研人员及政策制定者提供高效、可靠的数字化决策支撑。

## 技术架构

| 层级 | 技术栈 |
|------|--------|
| 前端 | HTML5 + CSS3 + JavaScript |
| 后端 | Python 3 + Django（MVC + ORM） |
| 数据库 | MySQL + DBeaver |
| 开发环境 | Windows 11 + VS Code |

## 核心功能

- **🔐 多角色权限系统**：四类用户身份注册与认证，权限分层管理，确保数据安全与操作合规
- **🌡️ 海洋环境数据监测**：按海域检索水质、水温、盐度等环境指标，支持时间维度筛选与数据增删改
- **🐟 渔业资源数据管理**：多维度检索各海域渔业资源种类与数量，海洋研究人员可实时维护数据
- **📋 渔业政策公示**：管理人员发布捕捞政策，全平台用户可查阅，实现信息对称
- **📝 海洋工作建议**：用户侧反馈机制，建议经审批流转至管理层，形成政策优化闭环
- **⚓ 渔业活动申请与审批**：从业人员提交活动申请，管理人员在线审核，全流程留痕可溯

## 数据库设计亮点

- 基于 E-R 模型完成 **17 张关系表**的规范化设计，达到第三范式
- 通过继承关系抽象用户实体，有效消除冗余，提升模式扩展性
- 针对高频查询字段建立主键索引与唯一索引，结合 NOT NULL 约束保障数据完整性
- 完整定义外键关联，覆盖海域、观测站、活动申请、政策、建议等核心业务实体

## 系统架构
```
前端页面 (8个) ──HTTP Request──▶ Django Views (API接口)

```

本项目将关系型数据库原理与实际渔业管理场景深度融合，完整覆盖需求分析、概念设计、逻辑设计、物理设计到系统实现的数据库工程全链路，是数据库系统原理课程设计的完整实践成果。
